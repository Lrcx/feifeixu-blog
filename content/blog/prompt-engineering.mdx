# Prompt Engineering 最佳实践

与 LLM 的对话质量很大程度上取决于你的提示词设计。掌握 Prompt Engineering，让 AI 更好地理解你的意图。

## 基础原则

### 1. 明确性

```text
❌ 帮我写一篇文章
✅ 请写一篇关于 AI Agent 的技术文章，目标读者是工程师，字数约 1000 字
```

### 2. 结构化

使用清晰的格式让模型理解任务：

```text
# 任务
分析以下代码的性能问题

# 输入代码
[代码内容]

# 输出格式
1. 问题列表
2. 优化建议
3. 修改后的代码
```

## 高级技巧

### Chain-of-Thought

引导模型展示推理过程：

```text
让我们一步步分析这个问题...
首先，我们需要理解...
然后，我们可以推导...
最终结论是...
```

### 角色扮演

```text
你是一位资深软件架构师，请从以下角度评审这个设计方案：
- 可扩展性
- 性能影响
- 维护成本
```

### 示例驱动

提供具体示例帮助模型理解：

```text
参考以下格式输出：
输入: "Hello"
输出: {"text": "Hello", "length": 5}

现在处理: "AI Agent"
```

## 常见陷阱

1. 过度复杂的指令
2. 缺少上下文
3. 模糊的输出要求
4. 忽略格式约束

## 工具推荐

- Prompt 模板库
- A/B 测试工具
- 版本管理系统
